# LLM (Large Language Model, 대규모 언어 모델)

> 방대한 텍스트로 학습해, **다음에 올 단어(토큰)를 예측**하는 방식으로 언어를 다루는 AI 모델.
> 한 줄 요약: **"지금까지의 문장을 보고, 다음에 올 말을 가장 그럴듯하게 이어 쓰는 기계."**

---

## 1. LLM이란?

LLM은 인터넷 문서·책·코드 등 **대규모 텍스트**로 학습한 신경망 모델이다.
핵심 동작은 의외로 단순하다. **"주어진 텍스트 다음에 올 토큰의 확률을 계산해서, 하나씩 이어 붙이는 것."**

- "Large(대규모)"인 이유: 학습 데이터가 방대하고, 모델 파라미터(내부 가중치)가 수십억~수천억 개에 달한다.
- 이 단순한 "다음 토큰 예측"을 엄청난 규모로 하다 보니, 번역·요약·코딩·추론 같은 능력이 **창발적으로(emergent)** 나타난다.

> 즉, LLM은 "정답을 아는 지식 DB"가 아니라 **"가장 그럴듯한 말을 확률적으로 생성하는 모델"** 이다.
> 이 성질을 이해하면 환각(hallucination) 같은 현상도 자연스럽게 설명된다.

---

## 2. 무엇을 할 수 있나

하나의 모델이 여러 작업을 별도 학습 없이 수행할 수 있다는 게 LLM의 큰 특징이다.

| 유형 | 예시 |
|------|------|
| **생성** | 글쓰기, 이메일 초안, 코드 작성 |
| **변환** | 번역, 요약, 문체 변경, 포맷 정리 |
| **분류·추출** | 감정 분석, 키워드 추출, 개체명 인식 |
| **질의응답** | 지식 질문, 문서 기반 Q&A (→ RAG) |
| **추론** | 단계적 사고, 수학·논리 문제 풀이 |
| **대화** | 챗봇, 상담, 역할극 |

## 3. 구성요소 / 핵심 원리

LLM 내부를 이해하는 데 필요한 핵심 부품들.

### 3.1 Token & Tokenization (토큰 / 토큰화)

- LLM은 글자를 그대로 다루지 않고, **토큰(token)** 이라는 단위로 쪼개서 처리한다.
- 토큰은 단어보다 작을 수 있다. 예: "먹었습니다" → `먹` + `었` + `습니다` 처럼 분해.
- 영어는 대략 1토큰 ≈ 0.75단어, 한국어는 글자당 토큰 수가 더 많은 편이다.
- **왜 중요한가**: 비용·속도·컨텍스트 길이가 모두 "토큰 수" 기준으로 계산된다.

### 3.2 Embedding (임베딩)

- 각 토큰을 의미를 담은 **벡터(숫자 배열)** 로 변환한다.
- 의미가 비슷한 토큰은 벡터 공간에서 가깝게 배치된다.
- 모델은 이 벡터들을 가지고 계산을 수행한다.

### 3.3 Transformer & Attention (트랜스포머 / 어텐션)

- 오늘날 LLM의 기반 구조는 **Transformer**(2017, "Attention Is All You Need")다.
- 핵심 메커니즘은 **Self-Attention(셀프 어텐션)**:
  각 토큰이 문장 안의 **다른 토큰들과 얼마나 관련 있는지**를 계산해, 문맥을 파악한다.
  - 예: "그는 은행에 갔다"에서 "은행"이 강가인지 금융기관인지를 주변 단어로 판단.
- 이 구조 덕분에 긴 문맥의 관계를 병렬로 효율적으로 처리할 수 있다.

### 3.4 Parameters (파라미터)

- 학습을 통해 조정되는 모델 내부의 **가중치(weight)** 값들.
- 파라미터 수(예: 7B = 70억, 70B = 700억)가 모델 규모를 나타내는 지표로 자주 쓰인다.
- 많다고 무조건 좋은 건 아니며, 데이터 품질·학습 방법도 성능을 크게 좌우한다.

### 3.5 학습 단계 (Training Stages)

| 단계 | 내용 | 목적 |
|------|------|------|
| **Pre-training (사전학습)** | 방대한 텍스트로 다음 토큰 예측을 대량 학습 | 언어·지식의 기본기 습득 |
| **SFT (지도 미세조정)** | (지시, 모범 답변) 쌍으로 추가 학습 | 지시를 따르도록 정렬 |
| **RLHF / 선호학습** | 사람 선호 피드백으로 강화학습 | 유용·안전·자연스럽게 다듬기 |

> Pre-training이 "언어를 배우는 단계"라면, SFT·RLHF는 "사람이 원하는 방식으로 답하도록 길들이는 단계"다.

## 4. 동작 흐름 (추론 시)

```
사용자 입력 (프롬프트)
   ↓ 토큰화
[토큰들] → 임베딩(벡터화)
   ↓
Transformer가 문맥을 계산
   ↓
다음 토큰의 확률 분포 산출
   ↓
샘플링으로 토큰 1개 선택 → 이어 붙임
   ↓ (선택된 토큰을 입력에 추가해 반복)
...토큰을 하나씩 생성...
   ↓
종료 조건 도달 → 최종 텍스트 출력
```

핵심은 **"한 번에 답을 다 만드는 게 아니라, 토큰을 하나씩 예측해 이어 붙인다(autoregressive, 자기회귀)"** 는 점이다.
그래서 답이 실시간으로 한 글자씩 흘러나오는 것처럼 보인다.

1. 토큰화: 입력 문장을 토큰 단위로 쪼갭니다. → 오늘 날씨가 정말

2. 벡터화 + 문맥 계산: 각 토큰을 벡터로 바꾸고, 트랜스포머(어텐션)가 문맥을 읽습니다.

3. 다음 토큰 확률 계산: "정말" 다음에 올 후보들의 확률을 뽑습니다. 예: 좋다(60%), 덥다(25%), 춥다(10%)…

4. 하나 선택(샘플링): 확률에 따라 하나를 고릅니다. → 좋다

5. 이어 붙이고 반복: 이제 입력이 "오늘 날씨가 정말 좋다"가 됩니다. 이걸 다시 1~4에 넣어 그다음 토큰을 예측합니다.

6. 종료: 문장이 끝났다는 신호(종료 토큰)가 나오거나 지정한 길이에 도달하면 멈추고, 지금까지 이어 붙인 걸 최종 답으로 내놓습니다.

## 5. 핵심 개념 자세히 보기

### 5.1 Context Window (컨텍스트 윈도우)

- 모델이 한 번에 **볼 수 있는 최대 토큰 수** (입력 + 출력 합산).
- 이 창을 넘어가면 앞부분을 "잊어버린다".
- 대화가 길어지거나 문서가 크면 이 한계에 부딪히며, RAG·요약 등으로 우회한다.

### 5.2 Temperature & Sampling (온도 / 샘플링)

다음 토큰을 확률 분포에서 어떻게 고르느냐를 조절하는 값.

| 설정 | 특징 | 적합한 작업 |
|------|------|-------------|
| **낮은 Temperature (예: 0~0.3)** | 가장 확률 높은 토큰 위주, 일관적·보수적 | 코드, 사실 답변, 추출 |
| **높은 Temperature (예: 0.8~1.2)** | 다양하고 창의적, 예측 불가 | 브레인스토밍, 창작 |

- 관련 파라미터: `top-p`(누적 확률 상위만 후보로), `top-k`(상위 k개만 후보로).

### 5.3 Prompt & Prompt Engineering (프롬프트)

- **프롬프트**: 모델에게 건네는 입력 텍스트 전체.
- 같은 모델이라도 프롬프트를 어떻게 쓰느냐에 따라 결과 품질이 크게 달라진다.
- 자주 쓰는 기법:
  - **Zero-shot**: 예시 없이 지시만.
  - **Few-shot**: 예시 몇 개를 함께 제공 (In-context Learning).
  - **Chain-of-Thought(CoT)**: "단계별로 생각해봐"처럼 추론 과정을 유도.
  - **역할 부여**: "너는 시니어 개발자야" 식으로 맥락 설정.

### 5.4 In-context Learning (문맥 내 학습)

- 모델 가중치를 바꾸지 않고, **프롬프트 안의 예시만으로** 새 작업을 수행하는 능력.
- Few-shot이 대표적. "학습(training)"이 아니라 "그때그때 예시 보고 따라 하기"에 가깝다.

### 5.5 Hallucination (환각)

- 사실이 아닌 내용을 **그럴듯하게** 만들어내는 현상.
- 원인: LLM은 "사실 검증"이 아니라 "그럴듯한 다음 토큰"을 생성하기 때문.
- 완화 방법: RAG(근거 문서 제공), 낮은 temperature, "모르면 모른다고 하라"는 지시, 출처 요구 등.

### 5.6 Fine-tuning (미세조정)

- 특정 도메인·태스크·말투에 맞게 모델을 **추가 학습**시키는 것.
- 지식 주입보다는 **행동·형식·톤**을 바꾸는 데 적합 (지식 최신화는 RAG가 유리).

## 6. LLM을 활용하는 방식

동일한 LLM을 목적에 따라 여러 방식으로 쓴다. 아래로 갈수록 손이 많이 간다.

| 방식 | 설명 | 언제 |
|------|------|------|
| **Prompting** | 프롬프트만 잘 써서 활용 | 가장 빠르고 저렴, 대부분의 시작점 |
| **RAG** | 외부 지식을 검색해 프롬프트에 결합 | 최신·사내·비공개 지식이 필요할 때 |
| **Fine-tuning** | 모델 자체를 추가 학습 | 특정 톤·포맷·전문 태스크 고정이 필요할 때 |
| **Agent / Tool use** | LLM이 도구(검색, 코드실행, API)를 호출하며 작업 수행 | 여러 단계의 자동화가 필요할 때 |

> 실무 순서 팁: **Prompting → RAG → Fine-tuning** 순으로, 필요할 때만 다음 단계로 올라가는 게 비용 효율적이다.

## 7. 장점과 한계

### 장점
- 하나의 모델로 다양한 작업 수행 (범용성).
- 자연어로 지시할 수 있어 진입장벽이 낮다.
- 재학습 없이 프롬프트·RAG만으로도 상당 부분 대응 가능.

### 한계
- **환각**: 틀린 내용을 자신 있게 말할 수 있다.
- **최신성**: 학습 시점 이후 지식은 모른다 (→ RAG로 보완).
- **컨텍스트 한계**: 한 번에 볼 수 있는 양에 제한이 있다.
- **비결정성**: 같은 질문에도 답이 매번 조금씩 달라질 수 있다.
- **비용·지연**: 큰 모델일수록 느리고 비싸다.
- **편향·안전성**: 학습 데이터의 편향을 그대로 반영할 수 있다.

## 8. 주요 용어 정리

| 용어 | 뜻 |
|------|-----|
| **Token (토큰)** | 모델이 처리하는 텍스트의 최소 단위 |
| **Parameter (파라미터)** | 학습으로 조정되는 모델 내부 가중치 |
| **Context Window** | 한 번에 볼 수 있는 최대 토큰 수 |

| **Transformer** | 현대 LLM의 기반 신경망 구조 |
| **Attention** | 토큰 간 관련성을 계산하는 핵심 메커니즘 |
| **Pre-training** | 대규모 텍스트로 하는 기본 학습 |
| **SFT** | 지시-답변 쌍으로 하는 지도 미세조정 |
| **RLHF** | 사람 선호 기반 강화학습 |
| **Hallucination** | 사실이 아닌 그럴듯한 생성 |
| **Prompt** | 모델에 건네는 입력 텍스트 전체 |
| **Inference** | 학습된 모델로 실제 답을 생성하는 과정 |

## 9. 평가 (Evaluation)

LLM 평가는 "무엇을 잘하는지"에 따라 방식이 다르다.

| 방식 | 설명 | 예시 |
|------|------|------|
| **벤치마크** | 표준 데이터셋으로 정량 측정 | MMLU(지식), HumanEval(코딩), GSM8K(수학) |
| **자동 지표** | 정답과 비교하는 점수 | BLEU·ROUGE(요약·번역), Perplexity |
| **LLM-as-a-Judge** | 강력한 LLM이 답변을 채점 | 개방형 답변 평가에 유용 |
| **사람 평가** | 사람이 직접 품질·선호 판단 | 최종 신뢰도 검증 |

> 자동 지표는 참고용이며, 개방형 작업일수록 사람/LLM 심사가 필요하다.
> 서비스 관점에선 정확도뿐 아니라 **지연시간·비용·안전성**도 함께 평가해야 한다.
